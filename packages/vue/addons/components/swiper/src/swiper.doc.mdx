# Swiper

`Swiper` 是 addons 中的轮播组件集合，面向轮播容器、分页器、导航按钮和滚动条等常见组合场景。该组件集基于 `swiper` 生态做二次封装，核心能力以底层库为主，并补充了 `SwiperPagination` 的 `autoplay-bullets` 扩展类型。当前不提供 `theme` 与 `ui` 配置入口，支持通过 `class`、结构化 `data-part` 以及 CSS 变量进行样式扩展。

### Props

`Swiper` 相关组件对底层 Swiper 配置存在“全部透传/部分透传”两种模式：

- `Swiper`：`SwiperProps` 继承 `SwiperOptions`，属于**全部透传**，并扩展 `tag`、`wrapperTag`、`modules`、`class`。
- `SwiperNavigationPrev` / `SwiperNavigationNext`：`SwiperNavigationProps` 继承 `Omit<NavigationOptions, 'enabled' | 'nextEl' | 'prevEl'>`，属于**部分透传**；`enabled` 与元素挂载由组件内部接管。
- `SwiperPagination`：`SwiperPaginationProps` 继承 `Omit<PaginationOptions, 'type' | 'enabled' | 'el'>`，属于**部分透传**；内部接管 `enabled`、`el`，并将 `type` 扩展为 `autoplay-bullets`。
- `SwiperScrollbar`：`SwiperScrollbarProps` 继承 `Omit<ScrollbarOptions, 'enabled' | 'el'>`，属于**部分透传**；`enabled` 与 `el` 由组件内部接管。

底层参数详情请参考 <a href="https://swiperjs.com/swiper-api">Swiper API</a>。

## Theme Configuration

`Swiper` 不提供 `theme` 配置能力。建议通过组件 `class`、`data-scope` / `data-part` 结构选择器与 CSS 变量进行样式控制。

---

## UI Configuration

`Swiper` 不提供 `ui` 或 `.ui.*` 配置能力。样式扩展建议：

- 对应组件传入 `class`（如 `Swiper`、`SwiperPagination`、`SwiperNavigationPrev`）。
- 基于 `index.css` 中的 `rui-swiper-*` 结构类与 `data-part` 节点做覆盖。
- 按需覆盖分页与滚动条的 CSS 变量（如 `--swiper-pagination-*`、`--swiper-scrollbar-*`）。

---

## 最佳实践

- 先按模块声明能力：需要分页/导航/滚动条时，优先将对应模块加入 `Swiper.modules`，再挂载对应子组件。
- 组合优先放在 `Swiper` 插槽内部：导航和分页建议优先放在 `container-start` / `container-end`，便于结构和定位统一。
- 外部控制场景使用 `swiper` 引用透传：当控制器组件需要放到容器外部时，通过 `ref` 获取实例后传给 `SwiperPagination`、`SwiperNavigation*`、`SwiperPrev/Next`。
- 使用 `SwiperPrev/Next` 自定义按钮时，基于 `default` 插槽的 `disabled` 状态同步禁用样式与点击反馈。
- 自动播放进度点建议使用 `SwiperPagination type="autoplay-bullets"`，并配合分页 CSS 变量统一进度外观。
- 需要较深样式定制时，优先走 `class + CSS 变量`，避免直接依赖底层 DOM 结构细节。
