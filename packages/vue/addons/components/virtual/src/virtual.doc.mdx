# Virtual

`Virtual` 是 addons 里的独立虚拟滚动组件集，用于长列表、动态高度列表和二维网格的按需渲染。组件核心由虚拟化库封装而来，并通过 `VirtualListItem` / `VirtualGridItem` / `VirtualInfiniteLoading` 组合扩展能力。当前不提供 `theme` 配置入口，支持部分 `ui` 样式覆盖。

## API

### Props

`VirtualList` 与 `VirtualGrid` 对 `@tanstack/vue-virtual` 属于“部分透传”：文档中仅列出常用和扩展项；底层虚拟器能力请参考
<a href="https://tanstack.com/virtual/latest/docs/framework/vue/vue-virtual">Vue Virtual 文档</a>。
其中 <code>enabled</code>、<code>count</code>、<code>getScrollElement</code> 由组件内部控制；<code>estimateSize</code>、<code>scrollToFn</code>、<code>observeElementRect</code>、<code>observeElementOffset</code> 作为扩展项对外开放。

## Theme Configuration

`Virtual` 不支持 `theme` 配置。替代方案：

- 使用 `class` 与 `ui.*` 覆盖局部样式。
- 引入并按需覆盖 `index.css` 中的结构化选择器（`data-scope` / `data-part`）。

## 最佳实践

- 组合优先：列表场景使用 `VirtualList + VirtualListItem`，网格场景使用 `VirtualGrid + VirtualGridItem`，无限加载作为附加节点放在默认插槽里。
- 动态高度优先开启 `VirtualListItem.dynamic`，并提供接近真实值的 `estimateSize`，减少滚动跳动。
- 处理加载态时统一从 `infinite` 事件回调维护状态，只在数据真正结束时调用 `complete()`。
- 网格调优先分开设置 `rowVirtualizerOptions` 与 `columnVirtualizerOptions`，不要只调单侧 `overscan`。
- 样式覆盖先用 `ui.*`，再用组件 `class`；需要结构级覆盖时再基于 `data-scope` / `data-part` 写样式。
- 若使用 TypeScript，`VirtualList` / `VirtualGrid` 的 `ui` 在类型声明与渲染路径存在差异时，应以渲染生效路径 `root` / `scroll` 为准并在业务侧补充类型声明。
