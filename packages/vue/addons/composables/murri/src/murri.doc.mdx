# Muuri

`Muuri`（目录名为 `murri`）是 addons 中基于 `muuri` 的拖拽排序 composable。它用于将列表节点初始化为可拖拽网格，并在拖拽移动时同步更新传入的列表数据。该能力属于 addons 独立封装，不提供 `theme` 或 `ui` 配置入口，样式扩展通过 CSS 类与样式文件完成。

### Props

`useMuuri` 对底层 `muuri` 配置属于“全部透传 + 默认增强”：

- 透传依据：`UseMuuriOpitons` 直接继承 `GridOptions`（以 interface/type 继承关系为准）。
- 内部接管：默认布局函数、默认拖拽参数、占位符创建与 `onMove` 时的列表重排同步。
- 其余底层参数见 <a href="https://docs.muuri.dev">Muuri 文档</a>。

## Theme Configuration

`Muuri` 不提供 `theme` 配置能力。可通过样式类（如 `rui-muuri-grid`、`rui-muuri-item`）与自定义 CSS 变量（如 `--muuri-placeholder-bg`）进行外观调整。

---

## UI Configuration

`Muuri` 不提供 `ui` 或 `.ui.*` 配置能力。建议通过容器 class 与样式文件中的结构类进行覆盖。

---

## 最佳实践

- 列表项结构保持稳定：每个项使用稳定 `key`，并保持 `rui-muuri-item` / `rui-muuri-item-content` 的层级，减少拖拽测量抖动。
- 优先使用手柄拖拽：默认 `dragHandle` 为 `.rui-muuri-handle`，建议为可交互区域提供明确手柄，避免整行误拖动。
- 需要监听行为时优先用 `onMove` + `onDragStart/onDragEnd` 组合：`onMove` 追踪顺序变化，拖拽事件追踪状态展示。
- 自定义布局前先评估默认纵向布局：当前默认布局按单列垂直堆叠并动态计算高度，适合任务流/卡片排序场景。
- 手动控制生命周期时配合 `start/stop`：在容器显隐或重建场景中可显式调用，避免残留实例。
