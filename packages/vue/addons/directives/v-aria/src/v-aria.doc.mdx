# v-aria

`v-aria` 是 addons 中的可访问性增强指令，用于给非原生可交互节点补齐 `tabindex`、`role` 与 `aria-*` 属性，并在需要时把 Enter 键映射为业务 action。该能力属于 addons 独立实现（非 Ark UI 对齐封装），当前不提供 `theme` / `ui` 配置入口，支持通过原生属性与类名扩展。

## Theme Configuration

`v-aria` 不提供 `theme` 配置能力。视觉层建议通过宿主元素 class 与样式系统控制。

---

## UI Configuration

`v-aria` 不提供 `ui` 或 `.ui.*` 配置能力。该指令只负责属性与键盘行为增强。

## 最佳实践

- 优先对“非原生交互元素”使用 `v-aria`，原生 `<button>`、`<a>` 应保留自身语义，避免重复声明 `role`。
- 若需要键盘触发行为，始终同时声明 `role` 与可读 `label`（或等价 `aria-label`），确保读屏可感知。
- 当交互节点嵌套较深时，优先使用 `pt: '.selector'` 精确命中目标，避免把 `tabindex` 写到外层容器。
- 对需要阻断冒泡的快捷交互，使用 `.stop` 并在 `action` 内只处理 Enter 语义，其他按键交给业务逻辑单独处理。
- 建议将复杂 ARIA 状态（如 `expanded`、`controls`、`describedby`）显式写在指令值中，避免在模板与脚本分散维护。
- 在频繁更新绑定值的场景中，保持 `action` 引用稳定，减少重复绑定带来的监听器重建成本。
