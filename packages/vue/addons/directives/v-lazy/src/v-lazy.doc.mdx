# v-lazy

`v-lazy` 是 addons 提供的图片/背景资源懒加载指令，用于在元素进入可视区域后再加载目标资源，并提供 `loading` / `error` 占位、滚动容器监听、数组源回退等能力。该能力属于 addons 独立实现（基于指令封装，不是 Ark UI 对齐组件），不提供 `theme` / `ui` 配置入口，可通过原生属性、类名与样式进行扩展。

`v-lazy` 为指令形态，不提供组件 Props。对外输入由“指令值（value）+ 指令参数（arg）+ 修饰符（modifiers）”组成。

## Theme Configuration

`v-lazy` 不提供 `theme` 配置能力。可通过宿主元素 class、CSS 变量与业务样式系统控制视觉表现（如占位尺寸、边框、背景样式）。

`v-lazy` 不提供 `ui` 或 `.ui.*` 配置能力。建议通过宿主元素属性与样式覆盖实现结构与表现定制。

## 最佳实践

- 图片资源优先使用对象值形式（`src/loading/error`）而不是仅传字符串，避免加载中与失败状态不可控。
- 当需要背景图懒加载时，明确使用 `v-lazy:background-image`，并在样式中预先定义 `bg-cover/bg-center` 等布局类，避免加载后布局跳变。
- 多候选资源（CDN 主备）建议使用 `src: string[]`，并把更高成功率地址放前面，减少回退链路耗时。
- 对滚动容器场景，优先保证容器可被稳定定位（`ref` 或 `id`），避免监听回退到错误的祖先滚动节点。
- 需要监听加载结果时，统一在插件层开启 `dispatchEvent`，再对目标元素订阅 `loaded/error`，便于埋点与告警收敛。
- 对频繁切换资源的列表项，可结合元素 `no-cache` 属性控制缓存策略，避免旧缓存命中影响实时性。
