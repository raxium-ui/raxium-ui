# Checkbox

Checkbox 是用于布尔选择与多选集合管理的复选框组件，支持单项勾选、混合态（`indeterminate`）以及组内批量选择。组件基于 Ark UI Checkbox 能力构建，根组件保留 Ark UI 的核心状态与表单语义，并扩展 `theme` 与 `ui` 配置用于样式定制。

## ARK UI Link

`Checkbox` 基于 Ark UI Checkbox Root 封装，`CheckboxProps` 继承 `CheckboxRootBaseProps`，根组件透传 Ark UI Root 能力。`CheckboxGroup` 复用 Ark UI Group 能力管理组内值集合。

- **相关文档**: [Ark UI Checkbox (Vue)](https://ark-ui.com/vue/docs/components/checkbox)
- **说明**: 文档主要覆盖 Raxium UI 暴露的常用能力与扩展项；Ark UI 原生的完整高级参数（如更细粒度 `ids`）建议参考官方文档。Raxium UI 额外扩展了 `label`、`theme` 与 `ui.*` 样式入口。

## 最佳实践

- 单个开关场景优先使用 `Checkbox` + `v-model:checked`，需要回显中间态时使用 `checked='indeterminate'`。
- 多项勾选场景优先用 `CheckboxGroup` 管理集合值，避免手动同步每个子项的数组状态。
- 表单提交流程中为项或组设置 `name` / `value` / `required`，确保浏览器原生语义与校验可用。
- 当业务需要统一风格时先用 `theme`（如 `size`、`skin`），仅在局部覆盖时再使用 `ui.*`，降低样式分散风险。
- 自定义图标时通过 `indicator` 插槽读取 `checkedState`，保证 `checked` 与 `indeterminate` 两种状态都可正确表达。
- 对禁用、只读、错误态分别使用 `disabled`、`readOnly`、`invalid`，不要用视觉样式替代语义状态。
