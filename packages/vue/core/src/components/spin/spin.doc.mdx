# Spin

`Spin` 是用于加载中状态反馈的遮罩组件，支持在局部容器内覆盖（`inline`）或全屏覆盖（`fullscreen`）。组件基于 Ark UI Factory 原语与 Raxium UI 主题体系构建，支持 `theme` 与 `ui` 分槽位样式配置，并可通过 `SpinProvider` 统一注入默认或自定义加载图标。

## ARK UI Link

`Spin` 不是 Ark UI 的一一对应业务组件（如 Dialog/Menu），而是基于 Ark UI Factory 原语（如 `ark.span`）封装的加载反馈组件。

- **相关文档**: [Ark UI Vue Factory](https://ark-ui.com/vue/factory)
- **透传边界**: `Spin` 的能力以 Raxium UI 自身实现为主（`show`、`mode`、`delay`、`theme`、`ui`、`SpinProvider`）；不依赖 Ark UI 独立 Spin 组件的 API。

## 最佳实践

- 页面内局部加载优先使用 `mode='inline'`，并保证父容器具备明确尺寸，避免遮罩高度抖动。
- 全屏加载仅用于阻塞式流程（如整页初始化/提交流程），避免频繁开关影响交互连续性。
- 需要平滑收尾时使用 `delay`，减少请求瞬时结束导致的闪烁；建议与真实网络耗时一起评估阈值。
- 统一图标风格时优先通过 `SpinProvider` 的 `icon` 插槽集中定义，避免在多个页面重复实现。
- 文案反馈通过 `text` 插槽提供（如 “Loading...”/“提交中...”），让用户区分“正在加载”与“无响应”。
- 视觉定制按“`theme` 优先、`ui` 补充”的方式处理：全局风格走 `theme`，局部差异再用 `ui.*`。
