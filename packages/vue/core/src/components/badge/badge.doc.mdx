# Badge

Badge 是轻量级状态标识组件，适用于标签、计数提示与角标提醒等场景。组件基于 Ark UI Factory 原语构建，支持 `default / secondary / dot` 变体、尺寸主题能力（`theme`），并可通过 `as` 切换为 `div` 或 `sup` 元素以适配不同布局语义。

## ARK UI Link

`Badge` 不对应 Ark UI 的独立组件，而是基于 Ark UI Factory 原语（`ark.div` / `ark.sup`）封装。组件会将业务扩展属性（如 `variant`、`theme`、`as`）消费后，把其余原生属性透传给底层元素。

- **相关文档**: [Ark UI Vue Factory](https://ark-ui.com/vue/factory)
- **透传边界**: 支持常规 HTML 属性透传（如 `id`、`data-*`、`aria-*`、原生事件等）；不提供 Ark UI 某个“Badge 组件”的完整 API 透传能力（因为 Ark UI 中无同名组件）。

## 最佳实践

- 纯文本或数字标识优先使用 `default`/`secondary` 变体，避免在 Badge 内放置复杂交互控件。
- 角标提醒场景使用 `variant="dot"` 并配合 `as="sup"`，让语义与布局更贴近通知角标。
- 需要统一视觉时，把尺寸和皮肤放入 `theme`，而不是在业务层分散写大量 class。
- 只做局部微调时优先使用 `class`，涉及整体样式体系时再通过 `theme.crafts` 覆盖。
- 如果需要点击、切换等交互行为，建议把交互放在外层组件；Badge 保持展示职责，减少状态耦合。
