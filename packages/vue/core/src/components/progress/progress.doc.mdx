# Progress

`Progress` 用于展示任务完成度或流程进度，支持线性进度条、圆形进度和弧形进度三种常见形态。组件基于 Ark UI Progress 构建，保留 Ark UI 的进度状态与可访问性语义，并在 Raxium UI 中扩展了 `theme`、局部 `ui` 覆盖与 `transfer` 视觉变体能力。

## ARK UI Link

`Progress` 组件族基于 Ark UI Progress 原语实现，属于“部分透传 + 扩展能力”：

- `Progress` 根组件透传 Ark UI Root 常用能力（如 `modelValue/defaultValue`、`min/max`、`orientation`、`formatOptions`、`translations` 等），并扩展 `theme` 与根 `class`。
- `ProgressLinear`、`ProgressCircle`、`ProgressArc` 使用 Ark UI Track/Range/Circle 系列原语封装默认样式，补充 `variant`、`ui.*` 与圆形尺寸扩展（支持 `number` 像素值）。
- `ProgressLabel`、`ProgressValueText`、`ProgressView` 直接从 Ark UI 导出，可与 `Progress` 组合使用。

Ark UI 文档： [Ark UI Progress (Vue)](https://ark-ui.com/vue/docs/components/progress-linear)

## 最佳实践

- 保持组合结构稳定：推荐固定使用 `Progress` 根组件承载状态，再按场景选择 `ProgressLinear`、`ProgressCircle` 或 `ProgressArc`。
- 状态管理优先单一数据源：需要与业务同步时使用 `v-model`（`modelValue` + `update:modelValue`），避免同时混用受控和非受控。
- 线性与圆形分工明确：流程条或列表进度优先线性；仪表盘或卡片概览优先圆形/弧形。
- `transfer` 变体用于“渐变迁移”反馈，建议搭配明确颜色变量，避免与主题主色冲突导致可读性下降。
- 纵向进度务必同时设置容器高度（如示例中的 `class="h-30"`），否则视觉上无法体现方向差异。
- 样式定制遵循“先 `theme` 后 `ui`”：全局规格走 `theme.size/skin`，局部节点微调走 `ui.track/ui.range` 或 `ui.circle*`。
