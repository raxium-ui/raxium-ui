# Switch

`Switch` 用于表达单一布尔开关状态（开/关），适合用于功能启停、偏好配置等场景。组件基于 Ark UI Switch 构建，支持受控/非受控状态、表单语义透传，并扩展了 Raxium UI 的 `theme` 与 `ui` 样式入口；同时提供 `SwitchLabel` 便于构建一致的标签结构。

## ARK UI Link

`Switch` 根组件通过 `useForwardProps` 将 Ark UI Switch Root 的常用能力透传到底层（如 `checked/defaultChecked`、`disabled`、`readOnly`、`required`、`name`、`value` 等），并在此基础上增加 Raxium UI 的 `theme` / `ui` 扩展。`SwitchLabel` 基于 Ark UI 的 Label 原语封装，主要用于标签样式与组合结构。

- Ark UI 文档：[Ark UI Switch (Vue)](https://ark-ui.com/vue/docs/components/switch)
- 透传边界：Ark UI Root 能力为主；文档重点说明 Raxium UI 额外扩展与常用字段。

## 最佳实践

- 单个布尔状态优先使用 `v-model:checked`，外部需要强制回写时再切换到完全受控模式。
- 在表单中使用时补齐 `name`、`value`、`required`，保证提交语义与校验一致。
- 标签内容建议放在 `SwitchLabel` 中，避免把说明文本直接塞进 `Switch` 根节点造成结构不清晰。
- 样式调整优先按层次处理：全局风格使用 `theme.size/crafts`，局部差异使用 `ui.root/control/thumb`。
- 禁用与只读要区分：禁止交互用 `disabled`，仅阻止编辑但保留可读语义用 `readOnly`。
