# Tabs

`Tabs` 用于在同一上下文中切换多个内容面板，适合设置页分组、信息分区与多视图切换。组件基于 Ark UI Tabs 构建，支持受控/非受控值、可滚动标签列表、方向切换等 Root 能力，并在 Raxium UI 中扩展 `theme` 主题配置与统一样式输出。

## ARK UI Link

`Tabs` 组件族（`Tabs`、`TabsList`、`TabsTrigger`、`TabsIndicator`、`TabsContent`）基于 Ark UI Tabs 原语封装，根组件通过 `useForwardProps` 透传 Ark UI Root 常用能力。

- **相关文档**: [Ark UI Tabs (Vue)](https://ark-ui.com/vue/docs/components/tabs)
- **透传边界**: Root 层能力（如值管理、方向、懒加载等）由 `Tabs` 透传；Raxium UI 额外提供 `theme` 与默认 crafts 样式。子组件不额外定义业务状态逻辑，主要负责结构与样式落位。

## 最佳实践

- 触发项与内容项始终使用同一套 `value` 命名，避免出现可点击但无对应面板的状态。
- 页面状态需要外部驱动时使用 `v-model` 受控模式；仅本地切换场景优先 `defaultValue` 简化状态管理。
- 标签数量较多时优先使用可横向滚动的 `TabsList`（参考 ScrollableList），并保留当前项可见性。
- 指示器是“可选增强层”，如需极简样式可不渲染 `TabsIndicator`，但建议通过选中态样式保留可识别反馈。
- 内容较重时结合 `lazyMount` 与 `unmountOnExit` 控制挂载策略，减少首次渲染与非激活面板开销。
- 样式定制遵循“先 theme 后 class”：全局风格走 `theme`，局部结构差异再用各子组件 `class` 做覆盖。
