# Menu

`Menu` 是一个复合菜单组件，用于承载命令选择、上下文菜单、单选/多选项等轻量操作入口。该组件基于 Ark UI Menu 构建，支持受控与非受控状态、分组与子菜单，并扩展了 Raxium UI 的 `theme` 与子组件 `ui` 样式覆盖能力。

## ARK UI Link

`Menu` 组件族与 Ark UI Menu 的关系如下：

- `Menu`、`MenuContent`、`MenuItem`、`MenuItemGroup`、`MenuCheckboxItem`、`MenuRadioItem`、`MenuRadioItemGroup`、`MenuTriggerItem` 为在 Ark UI 原语上的封装，保留核心交互并增加主题/样式能力。
- `MenuTrigger`、`MenuContextTrigger`、`MenuItemText`、`MenuItemIndicator`、`MenuContext` 直接从 Ark UI 导出，参数与行为以 Ark UI 为准。
- `MenuArrow` 基于 Ark UI Arrow/ArrowTip 组合封装，用于菜单箭头渲染。

- **相关文档**: [Ark UI Menu (Vue)](https://ark-ui.com/vue/docs/components/menu)

## 最佳实践

- 使用 `Menu` + `MenuTrigger` + `MenuContent` 作为固定骨架，再按需组合 `MenuItemGroup`、`MenuCheckboxItem`、`MenuRadioItemGroup` 扩展能力。
- 当状态需要与页面其他区域联动时，优先使用 `v-model:open`、`v-model:highlighted-value`、`v-model`（radio group）走受控模式。
- 需要埋点时，优先监听根组件 `select` 事件，统一记录 `value`，避免在每个菜单项重复绑定点击逻辑。
- 多选与单选场景分别使用 `MenuCheckboxItem` 与 `MenuRadioItemGroup`，不要混用同一组选项状态模型。
- 子菜单建议像 `Submenu` 示例那样独立 `Menu` 实例并配合 `positioning` 调整偏移，避免层级菜单遮挡。
- 视觉定制先用 `theme`（统一尺寸/风格），局部结构再用子组件 `ui.*` 覆盖，减少样式分散维护成本。
