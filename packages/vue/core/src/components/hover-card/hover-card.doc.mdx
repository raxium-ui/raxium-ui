# HoverCard

`HoverCard` 是一个“可交互的悬停浮层”组件，常用于展示用户摘要、快捷说明或次级操作入口。组件基于 Ark UI Hover Card 构建：根组件承接 Ark UI Root 状态机能力，`HoverCardContent` 负责内容结构与样式扩展，同时支持 Raxium UI 的 `theme` 与局部 `ui` 覆盖。

## ARK UI Link

`HoverCard` 组件族与 Ark UI Hover Card 对应，属于「部分透传 + Raxium UI 扩展」：

- **相关文档**：[Ark UI Hover Card (Vue)](https://ark-ui.com/vue/docs/components/hover-card)
- **说明**：`HoverCard` 根组件基于 Ark UI Root，透传 Root 常用参数（如 `open/defaultOpen`、`positioning`、`openDelay/closeDelay`、`lazyMount/unmountOnExit` 等），并扩展 `theme`；`HoverCardTrigger` 直接导出自 Ark UI 原语；`HoverCardContent` 在 Ark UI Content/Positioner 上增加结构化包裹与 `ui` 覆盖；`HoverCardArrow` 在 Ark UI Arrow 上增加 `theme/ui` 覆盖。

## 最佳实践

- 优先使用 `HoverCard` + `HoverCardTrigger asChild` + `HoverCardContent` 的标准组合，保证交互语义与可访问性一致。
- 内容区域建议保持“短信息 + 轻操作”，重交互场景改用 `Dialog/Drawer`，避免悬停态承载复杂流程。
- 需要外部状态联动时使用 `v-model:open` 与 `openChange`，并在业务层统一记录打开来源与行为埋点。
- 对于密集列表中的 HoverCard，建议调整 `openDelay/closeDelay`，减少误触和抖动感。
- 大幅定制样式时优先用 `theme` 控制整体视觉，再用 `ui.content/ui.inner/ui.arrowTip` 做局部覆盖，避免样式分散。
- 依赖异步数据时建议结合 `lazyMount` 与 `unmountOnExit`，减少首屏渲染开销并避免隐藏内容长期驻留 DOM。
