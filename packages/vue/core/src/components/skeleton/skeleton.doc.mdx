# Skeleton

Skeleton 用于在内容加载阶段提供结构占位，减少页面跳动并提前建立视觉预期。组件基于 Raxium UI 主题系统与 Ark UI Factory 原语实现，支持动画变体（`pulse` / `wave`）、形状变体（`rect` / `circle`），并支持通过 `theme` 扩展样式能力。

## ARK UI Link

`Skeleton` 不是 Ark UI 的独立业务组件（如 Dialog/Menu），而是基于 Ark UI Factory 原语封装的样式组件。其行为以样式占位为主，不包含复杂交互状态机。

- **相关文档**: [Ark UI Vue Factory](https://ark-ui.com/vue/factory)
- **说明**: 原生 HTML `div` 的通用属性可透传使用；`variant`、`shape` 与 `theme` 为 Raxium UI 扩展能力。

## 最佳实践

- 先用 `class` 明确占位块的宽高，再用 `shape` 表达结构语义（文本行、头像、卡片等），避免加载前后布局跳动。
- 文本占位优先使用多行短条组合（如 `h-4` + 不同宽度），比单条整宽占位更接近真实内容密度。
- 列表或信息流场景中，建议同一区域统一 `variant`，减少动画节奏不一致带来的视觉噪音。
- 需要骨架屏与真实内容叠放对比时，可使用默认插槽包裹内部结构，保持占位尺寸与目标内容一致。
- 全局风格统一优先通过 `theme` 管理；仅在局部页面差异化时再用 `class` 临时覆盖，降低维护成本。
