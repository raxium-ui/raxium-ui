# RadioGroup

`RadioGroup` 用于管理“单选集合”场景，适合筛选条件、配置项切换、设置面板等只允许选择一个值的交互。组件基于 Ark UI Radio Group 原语构建，支持受控/非受控值管理，并扩展了 `theme`、`variant` 以及 `ui` 样式入口。

推荐组合为 **`RadioGroup` → `RadioGroupLayout` → `RadioGroupItem`**：根组件负责组值与组级标签，布局组件控制选项排列方向，子项承载单个选项内容与指示器变体。

## ARK UI Link

`RadioGroup` 与 `RadioGroupItem` 基于 Ark UI Radio Group 封装，关系如下：

- `RadioGroup` 根组件基于 Ark UI Root，通过 `useRadioGroup` 承接组值状态，并在此基础上扩展 Raxium UI 的主题能力与组级 `label`。
- `RadioGroupItem` 基于 Ark UI Item 组合 `ItemControl/ItemText/ItemHiddenInput`，并扩展 `text`、`variant` 与 `ui` 样式入口。
- `RadioGroupLayout` 为 Raxium UI 扩展的布局容器，通过 `tvRadioGroup` 的 `layout` 变体控制选项纵向/横向排列，不参与 Ark UI 状态管理。
- Ark UI 原生完整参数（尤其高级可访问性与定位细节）可直接参考官方文档。

Ark UI 文档： [Ark UI Radio Group (Vue)](https://ark-ui.com/vue/docs/components/radio-group)

## 最佳实践

- 推荐结构：`RadioGroup` 包裹 `RadioGroupLayout`，再在布局内放置多个 `RadioGroupItem`；视觉排列用 `layout="stack" | "inline"`，键盘导航语义用根级 `orientation`。
- 同一组选项保持 `value` 类型一致（例如全字符串或全数字），避免比较与持久化时出现隐式转换问题。
- 普通表单优先用非受控（`defaultValue`）；需要与外部筛选状态联动时再切到受控（`v-model` + `update:modelValue`）。
- 组级说明使用 `RadioGroup` 的 `label` prop 或 `#label` 插槽；单项文案使用 `RadioGroupItem` 的 `text` prop 或默认插槽。
- 变体选择建议按语义区分：默认圆点用于“单选”，`checkbox` 变体用于与同页 checkbox 风格保持一致的视觉场景。
- 样式定制遵循“先 theme 后 ui”：全局风格与尺寸优先放在 `theme`，单项差异再通过 `ui.*` 覆盖。
- 需要品牌化图标或更复杂状态展示时，优先使用 `control/default` 作用域插槽，不要直接改写内部结构。
