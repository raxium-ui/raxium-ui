# RadioGroup

`RadioGroup` 用于管理“单选集合”场景，适合筛选条件、配置项切换、设置面板等只允许选择一个值的交互。组件基于 Ark UI Radio Group 原语构建，支持受控/非受控值管理，并扩展了 `theme`、`variant` 以及 `ui` 样式入口。

## ARK UI Link

`RadioGroup` 与 `RadioGroupItem` 基于 Ark UI Radio Group 封装，关系如下：

- `RadioGroup` 根组件基于 Ark UI Root，通过 `useRadioGroup` 承接组值状态，并在此基础上扩展 Raxium UI 的主题能力。
- `RadioGroupItem` 基于 Ark UI Item 组合 `ItemControl/ItemText/ItemHiddenInput`，并扩展 `label`、`variant` 与 `ui` 样式入口。
- Ark UI 原生完整参数（尤其高级可访问性与定位细节）可直接参考官方文档。

Ark UI 文档： [Ark UI Radio Group (Vue)](https://ark-ui.com/vue/docs/components/radio-group)

## 最佳实践

- 同一组选项保持 `value` 类型一致（例如全字符串或全数字），避免比较与持久化时出现隐式转换问题。
- 普通表单优先用非受控（`defaultValue`）；需要与外部筛选状态联动时再切到受控（`v-model` + `update:modelValue`）。
- 当选项较多或有分组文案时，使用 `orientation="vertical"` 提升可读性，并在根节点配合间距 class 控制密度。
- 变体选择建议按语义区分：默认圆点用于“单选”，`checkbox` 变体用于和同页 checkbox 风格保持一致的视觉场景。
- 样式定制遵循“先 theme 后 ui”：全局风格与尺寸优先放在 `theme`，单项差异再通过 `ui.*` 覆盖。
- 需要品牌化图标或更复杂状态展示时，优先使用 `control/default` 作用域插槽，不要直接改写内部结构。
