# Button

Button 是用于触发操作、提交表单或承载快捷动作的基础交互组件。组件基于 Ark UI 的 `ark.button` 原语封装，支持变体、尺寸、加载态和水波纹反馈，并扩展了 Raxium UI 的 `theme` 与 `ui` 配置能力。

## ARK UI Link

`Button` 基于 Ark UI Factory 的 `ark.button` 原语实现，底层是原生 `<button>` 能力。根组件会透传常见 HTML 按钮属性（如 `type`、`disabled`、`aria-*`、`data-*`），同时叠加 Raxium UI 的主题与样式扩展。

- **相关文档**: [Ark UI Vue Factory](https://ark-ui.com/vue/factory)
- **透传边界**: 该组件不是 Ark UI 独立业务组件（如 Dialog/Menu），而是基于 factory 原语封装。原生按钮属性可直接使用；`variant`、`ripple`、`loading`、`theme`、`ui` 等为 Raxium UI 扩展能力。

## 最佳实践

- 业务动作按钮优先使用 `default/normal/outline/text` 语义变体，避免仅靠颜色类名区分行为优先级。
- 若按钮状态由外部异步流程驱动，建议直接使用 `loading` + `disabled` 组合，防止重复提交与多次触发事件。
- 需要统一尺寸体系时，优先通过 `theme.size` 管理，而不是在页面里零散写 `class` 覆盖。
- 开启 `ripple` 时建议仅用于强调点击反馈的场景（如工具按钮），批量列表中应谨慎使用以控制视觉噪音。
- 若要深度定制样式，先用 `ui.root/ui.loading` 做局部覆盖；仅在需要系统级改造时再使用 `theme.crafts`。
- 需要将交互能力挂到自定义元素时使用 `asChild`，并确保子元素具备可访问性属性与可聚焦行为。
