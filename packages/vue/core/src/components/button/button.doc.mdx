# Button

Button 是用于触发操作、提交表单或承载快捷动作的基础交互组件。组件基于 Ark UI 的 `ark.button` 原语封装，通过 `tvButton` 提供 **变体（variant）** 与 **语义色（color）** 组合、尺寸与加载态，并支持水波纹反馈与 Raxium UI 的 `theme` / `ui` 配置。

## ARK UI Link

`Button` 基于 Ark UI Factory 的 `ark.button` 原语实现，底层是原生 `<button>` 能力。根组件会透传常见 HTML 按钮属性（如 `type`、`disabled`、`aria-*`、`data-*`），同时叠加 Raxium UI 的主题与样式扩展。

- **相关文档**: [Ark UI Vue Factory](https://ark-ui.com/vue/factory)
- **透传边界**: 该组件不是 Ark UI 独立业务组件（如 Dialog/Menu），而是基于 factory 原语封装。原生按钮属性可直接使用；`variant`、`color`、`ripple`、`loading`、`theme`、`ui` 等为 Raxium UI 扩展能力。扩展项以 `ButtonProps`（见下文 API）为准。

## Theme Configuration

`Button` 通过 `theme` 驱动 `tvButton` crafts（影响 root / loading 等样式位），并与全局 `useTheme` 默认值合并（如 `skin`、`surface`、`size`）。

根节点会输出 `data-variant`、`data-color`、`data-size` 等，便于主题层按属性选择样式。

## 最佳实践

- 用 **`variant` + `color`** 表达按钮形态与语义色，避免在业务里大量写死颜色类名；列表/表单场景可统一用 `solid` / `outlined` 与 `primary` / `default`。
- 图标按钮使用 **`variant="icon"`**，配合 `color` 保持与其它按钮一致的语义色体系。
- 异步提交场景使用 **`loading` + `disabled`**（或仅 `loading` 配合 `pointer-events-none` 样式），避免重复点击与多次触发 `click`。
- 需要统一尺寸时优先 **`theme.size`** 或 Provider 配置，而不是在每个页面单独写高度类。
- 开启 **`ripple`** 适合强调点击反馈（如工具栏）；在长列表或密集表单中应谨慎使用，降低视觉噪音。
- 深度定制样式时优先 **`ui.root` / `ui.loading`**；需要跨应用调整 `tvButton` 结构时再考虑 **`theme.crafts`**。
- 使用 **`asChild`** 将按钮行为合并到自定义元素时，请确保子元素具备可访问性与键盘可操作行为。
