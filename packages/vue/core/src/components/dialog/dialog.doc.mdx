# Dialog

Dialog 是一个复合弹窗组件，用于承载确认、表单编辑、详情展示等临时交互内容。组件基于 Ark UI Dialog 构建，支持受控/非受控状态、关闭来源追踪（`from`）、主题配置，以及头部/内容/底部的结构化组合。

### 函数式调用

<Canvas of={DialogStories.Functional} />

`dialog()` 可以在任何地方（组件内、路由守卫、utils 等）命令式地唤起弹窗。  
在组件 `setup` 内部，推荐改用 `useDialog()` 组合式函数——它会自动捕获当前 App 的 `appContext`，使弹窗内的内容可以访问 `vue-i18n`、`vue-router`、`pinia` 以及所有通过 `app.use()` / `app.component()` 注册的插件与全局组件。

```ts
// ✅ 推荐：组件内使用 useDialog()
const { dialog } = useDialog()

function handleClick() {
  dialog.open({ title: '…', content: '弹窗可访问 $t、useRouter() 等' })
}

// ✅ 可用：组件外（路由守卫、utils）手动传入 appContext
dialog({ title: '…' }, app._context)

// ⚠️ 旧用法（仍可用，但弹窗内无法访问主 App 的插件上下文）
dialog({ title: '…' })
```

## ARK UI Link

`Dialog` 组件族与 Ark UI 的关系分为两层：`Dialog`、`DialogTrigger`、`DialogCloseTrigger`、`DialogBackdrop` 通过类型接口继承 Ark UI 基础 Props；`DialogContent/Header/Body/Footer` 为 Raxium UI 的结构化扩展层。

- **相关文档**: [Ark UI Dialog (Vue)](https://ark-ui.com/vue/docs/components/dialog)
- **说明**: `DialogProps extends DialogRootBaseProps`，`DialogTriggerProps/DialogCloseTriggerProps/DialogBackdropProps` 分别继承 Ark 对应 BaseProps，属于**部分透传 + 扩展**模式：保留 Ark UI 交互能力，同时扩展 Raxium UI 的 `theme`（根组件）与 `from`（触发来源追踪）。

##### `useDialog()`

组合式函数，在组件 `setup` 内调用，自动捕获当前 App 的 `appContext` 并返回绑定了上下文的操作对象。推荐在组件内优先使用此 API。

返回值：`{ dialog: { open: (options: DialogOptions) => { close: () => void } } }`

```ts
const { dialog } = useDialog()
dialog.open({ title: '…' })   // 自动继承 vue-i18n、pinia、全局组件等
const instance = dialog.open({ title: '…' })
instance.close()              // 手动关闭
```

## 最佳实践

- 优先使用 `DialogHeader` + `DialogBody` + `DialogFooter` 组合，保证结构语义和一致样式。
- 需要追踪关闭来源时，为触发器和关闭器配置 `from`，并在 `openChange` 中统一记录。
- 复杂业务推荐使用受控模式（`v-model:open`），将弹窗状态与页面逻辑保持单一数据源。
- 内容较长时限制 `DialogBody` 高度并开启滚动，避免整页滚动与焦点管理混乱。
- 若需快速确认弹窗，优先使用组件内的 `useDialog()` 组合式 API（自动继承 App 插件上下文）；在组件外（路由守卫、utils）调用时使用 `dialog(options, appContext)`，并手动传入 `appContext`。
- 当自定义关闭区时，使用 `DialogContent` 的 `close` 插槽并显式设置 `show-close="false"`，避免重复关闭按钮。
