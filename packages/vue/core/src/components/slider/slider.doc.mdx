# Slider

`Slider` 是用于区间数值选择的滑动输入组件，支持单值与多 Thumb（范围）模式、水平/垂直方向、刻度标记与 Tooltip 展示。组件基于 Ark UI Slider 构建，根组件在透传 Ark UI Root 常用能力的同时，扩展了 Raxium UI 的 `theme` 与 `ui` 样式配置入口。

## ARK UI Link

`Slider` 组件族与 Ark UI Slider 的关系如下：

- `Slider` 根组件基于 `Slider.RootProvider + useSlider`，根 props 与事件透传 Ark UI Root 能力（如 `modelValue/defaultValue`、`min/max/step`、`orientation`、`origin`、`minStepsBetweenThumbs` 等）。
- `SliderThumb`、`SliderMarker` 基于 Ark UI 对应原语封装，保留原语属性并补充主题样式。
- `SliderTooltipThumb`、`SliderTooltipMarker` 通过组合 Ark Slider 与 Tooltip 提供“值提示”能力，属于 Raxium UI 扩展，不是 Ark UI 一一对应子组件。

- **相关文档**: [Ark UI Slider (Vue)](https://ark-ui.com/vue/docs/components/slider)

## 最佳实践

- 单值与范围统一使用数组值（如 `[30]`、`[20, 70]`），避免在业务层混用标量与数组造成状态分支复杂化。
- 需要联动外部筛选状态时使用受控模式（`v-model` + `update:modelValue`）；仅做初始展示时优先 `defaultValue`。
- 范围模式务必为每个 Thumb 明确设置 `index`，并在需要时配置 `minStepsBetweenThumbs` 防止手柄过度重叠。
- 垂直布局不要只改 `orientation`，同时调整容器高度与 `ui.control/ui.track/ui.range`，保证命中区与视觉一致。
- Tooltip 建议用于精确反馈场景：高频拖拽时可监听 `valueChange` 做实时预览，提交动作优先放在 `valueChangeEnd`。
- 样式定制按层次处理：全局风格优先 `theme`，局部结构优先 `ui`，仅在设计系统级改造时使用 `crafts` 覆写。
