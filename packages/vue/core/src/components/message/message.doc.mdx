# Message

`Message` 是基于 Ark UI Toast 能力封装的全局消息提示方案，适合反馈异步结果、状态变更与轻量通知。组件由 `Messager`（消息容器）与 `Message`（单条消息渲染）组合而成，支持函数式调用（`useMessage`）、主题配置（`theme`）与样式覆盖（`ui`）。

## ARK UI Link

`Message` 组件组基于 Ark UI Toast 构建，但属于“部分透传 + Rark 扩展”：

- `Messager` 内部使用 `createToaster` 与 `Toaster` 承载队列和容器能力。
- `Message` 基于 `Toast.Root` 渲染单条消息，透传 `ToastRootBaseProps`。
- 在 Ark 能力之上，额外扩展了 `theme`、`ui`、`showClose`、`render`、函数式 `useMessage()` 封装。

Ark UI 文档： [Ark UI Toast (Vue)](https://ark-ui.com/vue/docs/components/toast)

## Theme Configuration

`Message` 支持主题配置，入口包括：

1. `Messager.theme`：配置全局消息主题。
2. `Message.theme`：覆盖当前渲染节点主题参数。
3. `Message.options.theme`：单条消息级别覆盖，最终会与 `Message.theme` 合并，且后者被前者覆盖同名字段。

## 最佳实践

- 在应用根部至少挂载一个 `Messager`，再通过 `useMessage()` 在任意业务层触发消息，避免“无容器”告警。
- 需要完整自定义结构时优先用 `options.render`；仅替换图标、文本或关闭按钮时优先用 `Message` 的具名插槽，便于复用默认交互。
- 对异步请求统一使用 `message.promise()`，将 loading/success/error 生命周期消息绑定到同一流程，减少手动维护状态分支。
- 关闭按钮策略建议分层：全局默认用 `Messager.showClose`，单条特例再用 `options.showClose` 覆盖。
- 样式调整优先走 `theme`，需要精确节点覆盖时再用 `ui.*`，避免大量业务 class 直接侵入组件结构。
- `loading` 类型默认不显示关闭按钮；若业务需要可提供自定义 `render` 并配合 `MessageCloseTrigger` 管理关闭行为。
