# Toggle

`Toggle` 用于表达“可按下/取消按下”的二态按钮，适合静音、收藏、布局切换等即时状态开关场景。组件基于 Ark UI Toggle 构建，支持受控/非受控状态与表单语义透传，并扩展了 Raxium UI 的 `theme` 配置能力。

## ARK UI Link

`Toggle` 根组件基于 Ark UI Toggle Root 封装，常用状态和交互属性会透传到底层；Raxium UI 在此基础上增加了 `theme` 样式入口。若需要“多个 toggle 组成单选/多选组”，建议搭配 `ToggleGroup` 与 `ToggleGroupItem` 使用。

- Ark UI 文档：[Ark UI Toggle (Vue)](https://ark-ui.com/vue/docs/components/toggle)
- 透传边界：以 Ark UI Root 能力为主；本文重点说明 Raxium UI 的扩展能力与常用字段。

## 可访问性

- 组件基于 Ark UI，内置切换按钮语义与键盘交互能力（可通过键盘触发按下/取消按下）。
- 对仅图标的 toggle，建议提供可见文本或可访问名称（如 `aria-label`），避免语义缺失。
- 需要表达不可操作状态时使用 `disabled`，不要仅用视觉样式弱化交互。
- 与表单联动时建议补齐 `name` 与 `value`，确保状态可被原生提交流程识别。

---

## 最佳实践

- 单个独立状态优先使用 `Toggle`；多个互斥或集合选择优先交给 `ToggleGroup` 管理。
- 外部状态需要强同步时使用 `v-model:pressed` 受控模式；仅初始化默认值时使用 `defaultPressed`。
- 图标切换建议配合 `ToggleContext` 渲染状态差异，保持视觉反馈和真实状态一致。
- 样式定制遵循“先 theme 后 class”：共性走 `theme`，局部差异再用 `class` 覆盖。
- 禁用态应在业务逻辑层同时阻断动作触发，避免仅靠 UI 禁用造成状态不一致。
