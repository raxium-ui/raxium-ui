# Popover

`Popover` 是一个轻量浮层组件，用于承载补充说明、上下文操作与小型交互内容。组件基于 Ark UI Popover 原语构建，支持受控/非受控打开状态、定位策略，以及 Raxium UI 的 `theme` 与局部 `ui` 样式覆盖能力。

## ARK UI Link

`Popover` 组件族基于 Ark UI Popover 封装，根组件使用 `Popover.RootProvider` 与 `usePopover` 连接状态；`PopoverTrigger`、`PopoverCloseTrigger`、`PopoverTitle`、`PopoverDescription` 等由 Ark UI 直接导出。

- **相关文档**: [Ark UI Popover (Vue)](https://ark-ui.com/vue/docs/components/popover)
- **透传边界**: `Popover` 根组件继承 Ark UI Root 常用能力（如 `open/defaultOpen`、`positioning`、`modal`、`portalled`、`closeOnEscape` 等），并扩展 `theme`。`PopoverContent`/`PopoverClose`/`PopoverIndicator` 在 Ark 原语基础上增加了 Raxium UI 的样式封装与 `ui` 覆盖入口。

## 最佳实践

- 优先使用 `PopoverTrigger as-child` 挂载到已有按钮组件，避免额外包裹节点影响布局与可访问性。
- 需要业务联动时使用 `v-model:open` 受控模式，并以 `update:open` 作为状态单一入口。
- 内容中存在输入、开关等可交互元素时，建议在 `PopoverContent` 内使用上下文组件组织状态，减少外部事件耦合。
- 定位优先通过 `positioning.placement` 明确表达意图；同一页面内尽量保持方向一致，降低用户认知成本。
- 大范围视觉风格优先用 `theme`，局部结构差异再使用 `ui.*`，避免样式覆盖分散到业务页面。
- 在内容插槽中将 `PopoverArrow` 放在首位，可保持箭头与内容结构稳定，避免后续重排带来的视觉抖动。
