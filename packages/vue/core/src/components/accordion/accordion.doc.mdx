# Accordion

`Accordion` 用于将内容分成多个可独立展开/收起的区块，适合 FAQ、设置分组、长表单分节等场景。组件组包含 `Accordion`、`AccordionItem`（别名 `AccordionPanel`）、`AccordionTrigger`、`AccordionContent`；在 Ark UI Accordion 能力基础上，根组件扩展了 `theme`/`class`，触发器扩展了 `indicator` 与 `ui` 样式入口。

## ARK UI Link

`Accordion` 组件组基于 Ark UI 的 Accordion 构建，透传边界以 `props.ts` 中的接口继承为准（`AccordionRootBaseProps`、`AccordionItemBaseProps` 等），而不是仅依据 `useForwardProps` 的实现细节：

- `Accordion` 根组件继承 Ark Root 能力（如 `modelValue` / `defaultValue`、`multiple`、`collapsible`、`disabled`、`orientation` 等），并增加 `theme`、`class`。
- `AccordionItem`、`AccordionTrigger`、`AccordionContent` 分别对应 Ark 的 Item、ItemTrigger、ItemContent 原语；`AccordionTrigger` 额外提供 `indicator` 与 `ui`（Raxium 扩展）。
- `AccordionPanel` 与 `AccordionItem` 为同一组件的别名导出，便于与文档或其它设计系统命名对齐。

Ark UI 文档： [Ark UI Accordion (Vue)](https://ark-ui.com/vue/docs/components/accordion)

## 最佳实践

- **结构固定**：保持 `Accordion > AccordionItem > AccordionTrigger + AccordionContent` 层级，每个 `AccordionItem` 必须设置稳定的 `value`，便于受控与测试选择器引用。
- **单选与全关**：默认非 `multiple` 时若需允许全部收起，将根上 `collapsible` 设为 `true`；多选场景用 `multiple` 并配合 `defaultValue` / `modelValue` 管理数组。
- **受控同步**：与路由或表单状态绑定时使用 `modelValue` + `update:modelValue`，在 `valueChange` 中做埋点或副作用，避免双源状态。
- **禁用粒度**：整组禁用用根 `disabled`；仅某一节不可点用对应 `AccordionItem` 的 `disabled`。
- **长内容**：内容区仍可用常规布局与滚动容器；避免在触发器内塞入过多交互控件，以免与键盘/屏幕阅读器焦点模型冲突。
- **样式分层**：全局节奏用根 `theme.size`，局部差异用 `AccordionTrigger` 的 `ui` 或 `class`；需要整体换肤时再考虑 `crafts` 覆写以降低升级成本。
