# Pagination

`Pagination` 用于列表、表格等长数据场景的页码导航，支持上一页/下一页、首尾页、省略页码展示以及受控分页状态管理。组件基于 Ark UI Pagination 构建，在 Ark UI 根能力上扩展了 `theme` 与 `ui` 配置，并提供 `PaginationGoTo`、`PaginationPageSize` 等组合能力。

## ARK UI Link

`Pagination` 根组件基于 Ark UI Pagination Root 与上下文能力封装，常用分页状态与行为（如 `page`、`pageSize`、`count`、`siblingCount`、`type`、`getPageUrl`）会透传到底层；页码项、前后翻页与省略节点使用 Ark UI 对应原语渲染。

- **相关文档**: [Ark UI Pagination (Vue)](https://ark-ui.com/vue/docs/components/pagination)
- **说明**: `PaginationGoTo`、`PaginationGoToTrigger`、`PaginationPageSize` 是基于 Ark UI 上下文与 Raxium UI 组件（如 NumberInput/Select）构建的增强子组件，不是 Ark UI 一一对应子组件。

## 最佳实践

- 当分页状态需要与 URL、筛选条件或远程查询参数同步时，优先使用受控模式（`v-model:page` / `v-model:page-size`）。
- `type='link'` 场景建议始终提供 `getPageUrl`，保证页码项具备可分享/可回退的导航语义。
- 需要跳页输入时，优先通过 `PaginationGoTo + PaginationGoToTrigger` 组合，不要在外部重复实现页码边界校验。
- `PaginationPageSize` 建议与后端可接受的分页粒度保持一致，并通过 `sizes` 显式约束可选值。
- 大范围风格统一优先使用 `theme`，局部结构微调再使用 `ui.*`，避免在业务页面分散写样式覆盖。
- 列表数据量或过滤条件变化后，若当前页可能越界，应在业务层同步重置或校正 `page`，避免出现空页展示。
