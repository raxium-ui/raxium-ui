# Tree

`Tree` 用于展示层级结构数据，适合目录浏览、权限节点选择、配置树编辑等场景。组件基于 Ark UI Tree View 原语构建，支持展开/折叠、单选/多选、复选框勾选、焦点管理以及懒挂载策略，并扩展了 Raxium UI 的 `theme` 与 `ui` 样式配置能力。

## ARK UI Link

`Tree`、`TreeNode`、`TreeCheckboxNode` 基于 Ark UI Tree View 原语封装：

- `Tree` 根组件透传 Ark UI Root 常用能力（值管理、展开/聚焦、加载策略、选择模式等），并增加 `theme` 与 `ui` 入口。
- `TreeNode` / `TreeCheckboxNode` 基于 Node/Branch/Item 原语构建，补充默认渲染逻辑和可覆盖的渲染函数、插槽能力。
- 异步子节点加载能力（如 `loadChildren`、`loadChildrenComplete`、`loadChildrenError`）遵循 Ark UI 事件与回调模型。

Ark UI 文档： [Ark UI Tree View (Vue)](https://ark-ui.com/vue/docs/components/tree-view)

## 可访问性

- 组件基于 Ark UI，内置 tree role、键盘导航、焦点管理和层级语义。
- 建议为节点文本提供稳定可读标签，避免仅依赖图标表达含义。
- 禁用节点请通过 `isNodeDisabled`/数据字段统一声明，不要只做视觉置灰。
- 大数据场景建议搭配 `lazyMount`/`unmountOnExit`，降低初始渲染与非激活分支开销。

---

## 最佳实践

- 数据建模优先统一：用 `createTreeCollection` 收敛 `nodeToValue/nodeToString/nodeToChildren`，不要在视图层重复解析结构。
- 选中模型先定清楚：单选场景用 `selectionMode="single"` + `selectedValue`，批量选择场景优先 `TreeCheckboxNode` + `checkedValue`。
- 自定义数据结构时统一配置 `keyMap`，避免在每个节点组件里手写字段兼容逻辑。
- 状态追踪建议在根组件集中监听 `expandedChange/selectionChange/checkedChange`，减少节点级重复埋点。
- 大树性能优先策略是“懒挂载 + 局部展开”，并限制容器高度配合滚动区域。
- UI 定制遵循“先 theme 后 ui”：共性风格走 `theme`，局部差异再通过 `ui.*` 覆盖。
