# ToggleGroup

`ToggleGroup` 用于管理一组可切换按钮，适合文本样式工具栏、视图模式切换、筛选条件等场景。组件基于 Ark UI Toggle Group 原语构建，支持单选/多选模式、受控/非受控值管理，并扩展了 Raxium UI 的 `theme` 样式能力。

## ARK UI Link

`ToggleGroup` 与 `ToggleGroupItem` 基于 Ark UI Toggle Group 封装：

- `ToggleGroup` 根组件基于 Ark UI Root，通过 `useToggleGroup` 管理组状态，并增加 `theme` 入口驱动 Raxium UI crafts。
- `ToggleGroupItem` 基于 Ark UI Item，继承组上下文状态（如选中态、方向），并提供统一样式输出。
- Ark UI 原生完整参数（高级键盘交互、可访问性细节等）可参考官方文档。

Ark UI 文档：[Ark UI Toggle Group (Vue)](https://ark-ui.com/vue/docs/components/toggle-group)

## 可访问性

- 组件基于 Ark UI，默认支持组语义与键盘导航；方向为纵向时同样可使用方向键在组内切换焦点。
- 图标项建议补充可访问名称（如 `aria-label`），避免仅图形表达导致语义缺失。
- 禁用态请使用 `disabled` 属性，不建议只靠视觉样式模拟禁用。
- 需要表单联动时，请确保 `value` 稳定且类型一致，便于提交与回显。

---

## 最佳实践

- 互斥选择场景优先使用 `type="single"`；允许组合筛选时使用 `type="multiple"`。
- 仅初始化默认值时使用 `defaultValue`；需要与外部状态强同步时使用 `v-model`。
- 同一组内保持 `value` 语义稳定（例如都用字符串枚举），避免状态判断和持久化出现歧义。
- 图标项建议同时提供 tooltip 或文本说明，提升可学习性与可读性。
- 样式定制遵循“先 theme 后 class”：全局风格放在 `theme`，局部差异再通过 `class` 覆盖。
