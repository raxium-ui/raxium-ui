# TagsInput

`TagsInput` 用于管理“字符串标签集合”输入，适合关键词录入、技能标注、邮件地址批量输入等场景。组件基于 Ark UI Tags Input 构建，支持受控/非受控值管理、最大数量与校验策略、横向滚动与换行模式，并扩展了 Raxium UI 的 `theme` 与 `ui` 配置能力。

## ARK UI Link

`TagsInput` 与 `TagsInputItem` 基于 Ark UI Tags Input 原语封装，关系如下：

- `TagsInput` 根组件透传 `TagsInputRootBaseProps` 到 Ark UI Root，并扩展 `theme`、`inline` 和 `ui` 配置。
- `TagsInputItem` 基于 Ark UI Item 组合 `ItemPreview`、`ItemText`、`ItemInput`，并提供局部 `ui` 覆盖能力。
- 删除触发器等更细粒度部件可直接使用 Ark UI 子原语（示例中使用 `ArkTagsInput.ItemDeleteTrigger`）。

Ark UI 文档： [Ark UI Tags Input (Vue)](https://ark-ui.com/vue/docs/components/tags-input)

## 可访问性

- 组件基于 Ark UI，内置键盘交互语义：可通过键盘导航高亮标签，并在输入框与标签间移动焦点。
- 建议在表单中提供明确的可见标签或辅助文案，不仅依赖 placeholder 表达输入目的。
- 对 `invalid` 状态建议同步外部错误提示文本，避免仅靠颜色表达错误。
- 只读与禁用语义应区分：`readOnly` 保留可聚焦与可读，`disabled` 用于完全禁止交互。

---

## 最佳实践

- 先确定数据模式：需要和外部筛选或表单状态联动时用受控（`v-model`），仅初始化时优先非受控（`defaultValue`）。
- 校验策略集中到 `validate`：把去重、trim、最小长度、格式匹配统一放在一处，避免在多个事件里重复判断。
- 数量限制与反馈成对使用：设置 `max` 时同步监听 `valueInvalid`，为用户给出明确失败原因。
- 布局优先按内容密度选择：标签较长或较多时优先 `inline=true` 横向滚动，阅读优先场景可切换 `inline=false` 换行模式。
- 删除行为保持可见：在 `TagsInputItem` 默认插槽中放置删除触发器，避免只依赖键盘删除导致发现成本高。
- 样式定制遵循“先 theme 后 ui”：统一视觉走 `theme`，局部差异再用 `ui.*` 覆盖，降低后续升级成本。
