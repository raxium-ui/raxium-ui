# Tooltip

`Tooltip` 用于在悬停或聚焦时展示简短提示信息，适合图标按钮释义、字段补充说明和状态提示等轻量场景。组件基于 Ark UI Tooltip 原语构建，支持受控/非受控打开状态、延迟配置、定位策略，并扩展了 Raxium UI 的 `theme` 与局部 `ui` 样式覆盖能力。

## ARK UI Link

`Tooltip` 组件族与 Ark UI Tooltip 对应，属于“部分透传 + Raxium UI 扩展”：

- `Tooltip` 根组件基于 Ark UI Root，通过 `useTooltip` 管理状态，透传 Root 常用参数（如 `open/defaultOpen`、`disabled`、`positioning`、`openDelay/closeDelay`、`lazyMount/unmountOnExit`），并扩展 `theme`。
- `TooltipTrigger` 直接导出自 Ark UI 原语，交互行为与 Ark UI 保持一致（默认支持 hover/focus 触发）。
- `TooltipContent` 与 `TooltipArrow` 在 Ark UI 原语基础上增加 Raxium UI 主题与 `ui` 覆盖入口。

Ark UI 文档： [Ark UI Tooltip (Vue)](https://ark-ui.com/vue/docs/components/tooltip)

## 可访问性

- 组件基于 Ark UI，默认具备悬停与键盘聚焦触发语义；可通过键盘导航触发提示内容。
- 使用 `TooltipTrigger asChild` 挂载到交互元素时，需确保触发元素本身可聚焦（如 `button`、`a`）。
- Tooltip 仅用于“补充说明”，不要放置必须点击或必须阅读的核心流程信息。
- 禁用场景请使用 `disabled` 明确阻断提示，避免仅用样式隐藏导致语义与行为不一致。

## 最佳实践

- 简短文案优先：建议将 tooltip 文案控制在一到两行，复杂说明改用 `Popover` 或 `Dialog`。
- 优先非受控：纯提示场景使用默认触发（hover/focus），需要业务联动再切到 `v-model:open`。
- 点击触发建议走受控模式：当业务需要 click 打开时，使用外部状态控制 `open`，避免与默认 hover 语义混淆。
- 延迟按密度调节：密集列表建议设置 `openDelay/closeDelay`，减少误触和频繁闪动。
- 样式定制遵循“先 theme 后 ui”：统一外观用 `theme`，局部结构差异再通过 `ui.*` 覆盖。
- 位置策略显式化：在滚动容器或复杂层级中优先配置 `positioning.strategy='fixed'`，降低遮挡与抖动。
