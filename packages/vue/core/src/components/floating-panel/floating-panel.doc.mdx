# FloatingPanel

`FloatingPanel` 是一个可拖拽、可缩放、可分阶段（最小化/最大化/默认）的浮动面板组件，适用于工具窗、悬浮配置面板、辅助调试面板等场景。组件基于 Ark UI Floating Panel 能力封装，根组件透传 Ark UI 常用交互参数，并扩展了 Raxium UI 的 `theme`、初始透明度（`opacity`）、固定拖拽状态（`pinned`）与缩放触发轴（`resizeAxis`）能力。

## ARK UI Link

`FloatingPanel` 组件族与 Ark UI Floating Panel 对应，属于“部分透传 + Raxium UI 扩展”：

- 根组件会透传 Ark UI Root 常用参数与状态机能力（如尺寸、位置、打开状态、拖拽相关配置等）。
- `FloatingPanelTrigger` 直接导出自 `@ark-ui/vue`，保持 Ark UI 原语行为。
- `FloatingPanelContent`、`FloatingPanelHeader`、各 Trigger 子组件在 Ark UI 原语之上补充了默认样式与 `theme/ui` 覆盖入口。

Ark UI 文档： [Ark UI Floating Panel (Vue)](https://ark-ui.com/vue/docs/components/floating-panel)

## 最佳实践

- 推荐保持固定结构：`FloatingPanel` + `FloatingPanelTrigger` + `FloatingPanelContent` + `FloatingPanelHeader`，将交互按钮统一放在 `control` 插槽中。
- 若需要“窗口固定”能力，优先使用 `FloatingPanelPinTrigger` 与根组件 `pinned` 初始值配合，不要手动改写拖拽逻辑。
- 透明度调节建议通过 `FloatingPanelOpacityTrigger` 的作用域插槽驱动，使用 `setOpacity` 统一更新，避免外部多处状态源冲突。
- 需要精细控制缩放交互时，优先用 `resizeAxis` 切换内置策略；仅在 `custom` 场景下自行渲染自定义缩放触发器。
- 处理复杂业务时建议监听 `openChange`、`stageChange`、`sizeChangeEnd`、`positionChangeEnd`，用于埋点和状态持久化。
- 面板内容较长时，建议在内容区内部自行处理滚动与虚拟列表，避免将大体量渲染压力放在拖拽/缩放交互层。
