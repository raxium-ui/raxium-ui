# RatingGroup

`RatingGroup` 用于评分场景（如满意度、服务质量、内容打分），支持受控/非受控值管理、悬浮预览、半星评分和禁用/只读状态。组件基于 Ark UI Rating Group 构建，并扩展了 Raxium UI 的 `theme` 与 `ui` 样式入口；同时暴露 `$api` 便于编程式设置与清空评分。

## ARK UI Link

`RatingGroup` 与 `RatingGroupItem` 基于 Ark UI Rating Group 原语封装：

- 根组件透传 Ark UI Root 常用能力（如 `count`、`disabled`、`readOnly`、`allowHalf`、`defaultValue`、`modelValue` 等），并增加 `theme`、`ui.root`、`ui.control`。
- 子项组件基于 Ark UI Item 封装，增加了 `theme`、`ui.root`、`ui.indicator`、`ui.icon` 以及默认星形渲染。
- 根组件通过 `defineExpose` 暴露 `$api`（来自 Ark `useRatingGroup`），可用于 `setValue`、`clearValue` 等编程式调用。

Ark UI 文档： [Ark UI Rating Group (Vue)](https://ark-ui.com/vue/docs/components/rating-group)

## 最佳实践

- 业务需要和表单状态同步时，优先用 `v-model` 受控模式；纯展示或轻交互场景可用 `defaultValue` 简化状态管理。
- 有“半分”需求时开启 `allowHalf`，并确保评分文案或提示支持小数表达（如 3.5 分）。
- 只读展示与禁用交互语义不同：展示历史评分用 `readOnly`，不可操作状态用 `disabled`。
- 需要记录用户评分意图时，同时监听 `valueChange` 与 `hoverChange`，区分“悬浮预览”和“最终提交”。
- 样式定制先用 `theme` 做全局一致性，再用 `ui` 做局部覆盖；大范围改造时才使用 `crafts` 覆写。
- 需要外部按钮联动评分时，使用 `ref` 调用 `$api.setValue()` / `$api.clearValue()`，避免在多个地方重复维护评分逻辑。
