# Toast

`Toast` 是基于 Ark UI Toast 封装的全局轻提示方案，适合异步反馈、状态提醒与轻量操作提示。组件由 `ToasterManager`（容器管理）、`Toaster`（队列与定位）和 `Toast`（单条渲染）组合，支持 `useToast()` 编程式调用、主题配置（`theme`）与节点样式覆盖（`ui`）。

## ARK UI Link

`Toast` 组件组基于 Ark UI Toast，属于“部分透传 + Rark 扩展”：

- `Toaster` 使用 `createToaster` 与 Ark `Toaster` 承载队列、时长与布局能力。
- `Toast` 基于 Ark `Toast.Root` 渲染单条提示，透传 `ToastRootBaseProps`。
- 在 Ark 能力之上扩展了 `theme`、`ui`、`render`、`ToasterManager` 与 `useToast()` 便捷方法。

Ark UI 文档： [Ark UI Toast (Vue)](https://ark-ui.com/vue/docs/components/toast)

## Theme Configuration

`Toast` 支持分层主题配置：

1. `Toaster.theme`：容器级默认主题（影响当前容器内所有提示）。
2. `Toast.theme`：当前渲染节点主题。
3. `Toast.options.theme`：单条提示级覆盖，优先级最高。

## 可访问性

- 组件基于 Ark UI，内置 toast 语义与焦点管理；建议保留默认结构，避免仅用纯文本容器替代。
- 使用 `duration: Infinity` 时应提供可见关闭入口（如 `ToastCloseTrigger`），避免键盘与读屏用户无法结束提示。
- 关键流程建议使用 `title + description` 双层信息，保证读屏上下文完整，不仅依赖图标表达状态。
- 多位置容器并存时，建议保持统一 `placement` 策略，降低焦点跳转与视觉定位成本。

---

## 最佳实践

- 应用根部建议挂载 `ToasterManager`（或至少一个 `Toaster`），再通过 `useToast()` 在任意业务层触发提示，避免无容器告警。
- 状态型反馈优先用 `toast.success/error/info/warning/loading`，减少重复传入 `type` 的样板代码。
- 异步链路优先使用 `toast.promise()`，将 loading/success/error 生命周期绑定到同一流程。
- 需要业务操作（撤销、重试、跳转）时，在 `render` 或 `close`/`inner` 插槽中组合 `ToastActionTrigger` 与 `ToastCloseTrigger`。
- 样式定制遵循“先 `theme` 后 `ui`”：全局一致性走 `theme`，节点级微调再用 `ui.*`。
- `loading` 类型默认不展示关闭按钮；若业务必须手动关闭，建议用自定义 `render` 显式提供关闭触发器。
