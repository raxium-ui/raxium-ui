# Select

`Select` 是用于单选或多选场景的下拉选择组件，适用于表单输入、筛选器和配置项选择。组件基于 Ark UI Select 能力构建，支持受控/非受控状态、高亮项控制、分组选项、可清空触发器，并扩展了 Raxium UI 的 `theme` 与局部 `ui` 配置能力。

## ARK UI Link

`Select` 组件族（`Select`、`SelectTrigger`、`SelectContent`、`SelectItem`、`SelectItemGroup`）基于 Ark UI Select 原语封装：

- 根组件在 Ark UI Root 常用能力之上扩展了 Raxium UI 的 `theme` 与样式体系。
- 子组件复用 Ark UI 对应部件并补充默认样式、可选插槽和少量增强属性（如 `SelectTrigger.clearable`、`SelectItemGroup.label`）。
- `SelectValue` 用于值展示区域封装，默认渲染 Ark UI 的 `Select.ValueText`。

- **相关文档**: [Ark UI Select (Vue)](https://ark-ui.com/vue/docs/components/select)

## 最佳实践

- 选项数据优先统一使用 `createListCollection` 生成 `collection`，避免手写数组导致分组与禁用逻辑分散。
- 普通表单场景优先使用非受控（`defaultValue`），仅在需要和外部状态同步时再切换受控（`v-model`、`v-model:open`）。
- 多选场景建议配合 `SelectTrigger.clearable`，为用户提供显式清空入口，降低重复点击成本。
- 需要追踪键盘导航或高亮行为时，监听 `highlightChange` 并按需使用 `v-model:highlighted-value` 做受控联动。
- 分组列表优先使用 `SelectItemGroup`，不要在选项文本中混合“组名 + 选项名”来模拟分组，避免可访问性与样式维护问题。
- 样式定制遵循“先 theme 后 ui”：统一视觉走 `theme`，局部覆盖再用 `SelectContent.ui.*` / `SelectItemGroup.ui.*`，减少升级时的样式破坏风险。
