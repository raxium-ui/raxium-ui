# Hotkey

`Hotkey` 用于在页面中采集和展示快捷键组合（如 `Ctrl + Shift + K`），适合设置中心、编辑器偏好等场景。组件基于 Ark UI `factory` 原语（`ark.div` / `ark.input`）构建，不对应 Ark UI 的独立业务组件；在原语能力上扩展了热键解析、删除清空、IME 输入保护、`theme` 与 `ui` 样式配置入口。

## ARK UI Link

`Hotkey` 不是 Ark UI 官方的一一对应组件，而是使用 Ark UI Vue `factory` 原语自行封装的输入组件：

- 透传边界：根节点与输入框由 `ark.div` / `ark.input` 渲染，保留 Ark 原语语义能力。
- 扩展能力：增加热键采集逻辑（按键组合排序、删除清空、冲突兜底）、`theme`（`tvHotkey`）和 `ui`（`ui.root` / `ui.input`）配置。

Ark UI 参考： [Ark UI Vue Factory](https://ark-ui.com/vue/factory)

## 最佳实践

- 优先使用 `v-model:hotkey`：将热键字符串作为单一事实源，便于与设置持久化逻辑对接。
- 区分只读与禁用：可查看但不可编辑用 `readonly`，不可交互且弱化视觉反馈用 `disabled`。
- 将业务监听放在 `change`：持久化或联动配置建议基于 `change` 回调，拿到标准化 `keycodes` 与最终 `hotkey` 文本。
- 将删除行为单独处理：监听 `cancel`，可针对清空动作做二次确认或回退逻辑。
- IME 环境提前提示：在中文输入法场景，建议同时监听 `error` 并给用户可见提示，降低录入失败困惑。
- 样式定制分层处理：全局风格优先用 `theme`，局部样式差异再用 `ui.root/ui.input`，减少后续升级冲突。
