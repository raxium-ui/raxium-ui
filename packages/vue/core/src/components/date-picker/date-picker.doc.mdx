# DatePicker

`DatePicker` 用于单日期与日期范围选择，支持日/月/年多视图切换、受控状态管理、禁用日期规则与事件监听。组件底层能力来自 Ark UI Date Picker，在 Raxium UI 中额外集成了 `theme` 主题能力与视图样式扩展，适用于表单录入、筛选面板与时间范围查询场景。

## ARK UI Link

`DatePicker` 与 Ark UI 的关系是**部分透传 + Rark 扩展**：

- 根组件基于 Ark UI 的 `useDatePicker` 与 `DatePicker.RootProvider`，核心交互模型（值、视图、打开状态、可用日期判断）沿用 Ark UI。
- `DatePicker` 根组件主要透传 `DatePickerRootBaseProps`，并新增 Rark `theme` 能力（包括 `tvDatePicker` crafts 覆盖）。
- `DatePickerControl` / `DatePickerContent` / `DatePicker*View` 在 Ark 原语外增加了 Rark 默认结构与主题样式封装，不属于完全无差异透传。

- Ark UI 文档：[Ark UI Vue Date Picker](https://ark-ui.com/vue/docs/components/date-picker)

## 最佳实践

- 优先保持 `DatePicker + DatePickerControl + DatePickerContent + DatePicker*View` 的组合结构，避免直接跳过控制区导致可访问性和交互语义不完整。
- 表单场景建议使用非受控或仅受控 `modelValue`；只有在需要外部驱动弹层开关时再引入 `v-model:open`。
- 需要埋点时，优先监听 `valueChange` 和 `openChange`，并结合 `update:modelValue` 做业务状态同步。
- 范围选择场景建议显式设置 `selectionMode="range"`，并在 UI 上通过 `DatePickerValueText` 或外部文本同步展示开始/结束日期。
- 对未来日期、节假日、库存日期等限制，统一通过 `isDateUnavailable` 处理，保证禁用逻辑与键盘导航行为一致。
- 当视觉规范要求强定制时，先用 `theme.size` 与 `crafts` 调整整体，再按需叠加 `class` 做局部覆盖，避免大量硬编码样式散落在业务层。
