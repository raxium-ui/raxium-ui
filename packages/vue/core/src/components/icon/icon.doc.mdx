# Icon

`Icon` 是基于 Iconify 的通用图标组件，用于在按钮、表单、列表和状态提示中统一渲染图标。组件不对应 Ark UI 的独立业务组件，但在 Raxium UI 体系内提供了 `theme` 入口（`tvIcon`）与 `class` 样式覆写能力，便于和全局主题保持一致。

## ARK UI Link

`Icon` 不对应 Ark UI 官方组件，底层渲染由 `@iconify/vue` 的 `Icon` 完成；组件不继承 Ark UI 的 Props 接口，不属于 Ark UI 组件族的“完全透传”场景。

- Ark UI 参考： [Ark UI Vue Utilities](https://ark-ui.com/vue/docs/utilities)
- Iconify 参考： [Iconify for Vue](https://iconify.design/docs/icon-components/vue/)

## Theme Configuration

`Icon` 支持通过 `theme` 参与 Raxium UI 统一主题系统，并由 `tvIcon` 生成最终 class。`tvIcon` 当前默认基线样式为 <code>size-[1lh]</code>，可通过 `theme.crafts` 覆写。

## 最佳实践

- 图标来源统一使用 Iconify 命名空间（如 `lucide:*`），避免团队内多套图标源并行导致视觉不一致。
- 需要与文本行高对齐时，优先通过字体级别（如父级 `text-sm/text-base`）控制尺寸，利用 `tvIcon` 的 `size-[1lh]` 保持对齐。
- 优先使用 `class` 完成颜色、动画、描边等单点样式调整；多页面复用样式再考虑沉淀到 `theme.crafts`。
- 组件不提供事件与插槽，交互语义（点击、提示、可访问性）应由外层按钮/链接等语义组件承载。
- 若业务需要固定像素尺寸或其它 Iconify 参数，先在项目中约定统一传参策略，避免不同页面出现图标比例漂移。
