# Overlay

`Overlay` 用于统一管理“运行时创建”的浮层实例（如通过函数式方式打开的 Dialog）。当前能力由 `OverlayProvider` 与 `useOverlay()` 组合提供：`OverlayProvider` 负责挂载与生命周期回收，`useOverlay()` 负责创建、打开、关闭与更新浮层实例。该能力参考 Nuxt UI Overlay 方案实现，并可与基于 Ark UI 的弹层组件配合使用。

## ARK UI Link

`Overlay` 本身不是 Ark UI 的直接组件封装，而是运行时浮层管理能力：

- 浮层内容组件通常会使用 Ark UI 组件（例如示例里的 `Dialog`）。
- `OverlayProvider` 通过动态组件渲染与事件桥接（`open-change`、`exit-complete`）驱动实例状态。
- `useOverlay()` 提供创建/打开/关闭等编排能力，不直接对应 Ark UI 某个单体组件 API。

可参考 Ark UI Dialog 文档了解常见承载组件： [Ark UI Dialog (Vue)](https://ark-ui.com/vue/docs/components/dialog)

## 最佳实践

- 在应用根部放置单个 `OverlayProvider`，集中管理运行时浮层，避免多 provider 导致状态分散。
- 需要复用同一浮层实例时，先调用一次 `create()` 并复用返回实例；高频临时弹层可配合 `destroyOnClose` 减少残留。
- 使用 `open(props)` 时仅传递本次变化字段，其余字段由 `originalProps` 合并，避免每次重复构建完整参数。
- 需要等待用户结果（如确认/取消）时，优先 `await` `open()` 返回 Promise，而不是在页面层维护额外中间状态。
- 将样式与主题配置放在承载组件（如 `Dialog`）内部，`Overlay` 只负责生命周期与状态编排，保持职责单一。
