# ScrollArea

`ScrollArea` 用于在固定容器内承载可滚动内容，支持垂直/水平滚动条组合与角落补位（`Corner`）。组件基于 Ark UI Scroll Area 构建，根组件透传 Ark UI Root 能力，并扩展 `theme` 与 `ui` 配置用于样式定制。

## ARK UI Link

`ScrollArea`、`ScrollAreaScrollbar`、`ScrollAreaCorner` 基于 Ark UI Scroll Area 原语封装：

- `ScrollArea` 根组件会将 `ScrollAreaRootBaseProps` 透传给 Ark UI Root（通过 `useForwardProps`）。
- `ScrollAreaScrollbar` 与 `ScrollAreaCorner` 分别封装 Ark UI 的 `Scrollbar` 与 `Corner`，并补充 `theme/ui` 样式入口。
- 组件额外提供滚动事件透出（`scrollstart` / `scrollend` / `scroll`）与基于内容尺寸的滚动条显示控制。

- **相关文档**: [Ark UI Scroll Area (Vue)](https://ark-ui.com/vue/docs/components/scroll-area)

## 最佳实践

- 推荐按 `ScrollArea + ScrollAreaScrollbar (+ ScrollAreaCorner)` 的固定结构组合，避免遗漏滚动条节点导致“可滚动但无反馈”。
- 单轴滚动场景仅保留一个 `ScrollAreaScrollbar`，双轴场景再补充 `orientation="horizontal"` 与 `ScrollAreaCorner`。
- 业务上需要监听滚动行为时，优先使用 `scrollstart/scroll/scrollend` 在根组件统一收敛事件，不在内容节点重复绑定。
- 全局风格差异优先用 `theme.size` 或 `theme.crafts`，局部微调再使用 `ui.*`，降低样式维护成本。
- 对超大内容建议限制容器尺寸并把布局交给 `Viewport`，不要依赖外层页面滚动替代局部滚动区域。
- 当内容宽高不确定时，先使用 `BothAxes` 场景验证横纵滚动与角落显示，再落地到业务页面。
