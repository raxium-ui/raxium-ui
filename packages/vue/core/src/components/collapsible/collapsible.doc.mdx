# Collapsible

`Collapsible` 用于在有限空间中分段展示信息，支持展开/收起状态切换。该组件组包含 `Collapsible`、`CollapsibleTrigger`、`CollapsibleContent`，并提供面向长文本场景的 `ReadMore` 封装；在 Ark UI 原语能力基础上，扩展了 `theme` 与部分 `ui` 样式配置入口。

## ARK UI Link

`Collapsible` 组件组基于 Ark UI 的 Collapsible 构建，但不是“全量无差别透传”：

- `Collapsible` 根组件将 Ark UI Root 常用能力（如 `open`、`defaultOpen`、`disabled`、`collapsedHeight`）透传，同时增加 `theme/class` 扩展。
- `CollapsibleTrigger`、`CollapsibleContent` 基于 Ark 对应原语封装，补充了默认样式与可选 UI 覆盖点。
- `ReadMore` 是基于上述三个组件组合出的业务便捷组件，不是 Ark UI 的一一映射子组件。

Ark UI 文档： [Ark UI Collapsible (Vue)](https://ark-ui.com/vue/docs/components/collapsible)

## 最佳实践

- 保持结构稳定：推荐固定使用 `Collapsible > CollapsibleTrigger + CollapsibleContent`，避免在运行时频繁增删子节点影响可访问性语义。
- 状态管理先选非受控：仅在需要和外部业务状态同步时使用 `open` 受控模式，并监听 `update:open` 或 `openChange` 做单向数据更新。
- 触发器文案可读优先：当使用 `indicator` 插槽自定义图标时，同时保留可理解文本，避免只依赖视觉符号。
- 长内容优先用 `ReadMore`：通过 `collapsedHeight` 与 `text.more/text.less` 统一阅读体验，不建议重复造轮子实现测量与折叠逻辑。
- 样式定制分层处理：大范围风格用 `theme`，局部差异用 `ui`；当需要大幅改造时再使用 `crafts` 覆写，减少后续升级成本。
