# Input

`Input` 用于采集单行文本或数字输入，适合表单、筛选栏、搜索框等场景。组件基于 Ark UI Vue `factory` 原语（`ark.div`）构建输入容器，在原生输入行为之上补充了 `clearable` 清空能力、`prefix/suffix` 插槽、`theme` 主题参数与 `ui` 局部样式覆盖入口。

## ARK UI Link

`Input` 不是 Ark UI 官方一一对应的业务组件，而是基于 Ark UI Vue `factory` 原语封装的输入组件：

- 透传边界：外层容器使用 `ark.div`，保留 Ark 原语语义能力。
- 扩展能力：增加 `clearable`、`prefix/suffix` 插槽、`theme`（`tvInput`）与 `ui`（`ui.root` / `ui.input` / `ui.clearable`）配置。
- 原生输入属性：通过组件自身显式 props 提供常用输入能力（如 `placeholder`、`maxlength`、`disabled`、`readonly`）。

Ark UI 参考： [Ark UI Vue Factory](https://ark-ui.com/vue/factory)

## 最佳实践

- 优先确定受控策略：需要外部统一管理值时使用 `v-model`（`modelValue` + `update:modelValue`），仅初始化值时使用 `defaultValue`。
- 区分 `readonly` 与 `disabled`：可查看不可编辑用 `readonly`，完全不可交互且弱化样式反馈用 `disabled`。
- 清空能力配合输入反馈：启用 `clearable` 时建议同时监听 `input` 或 `update:modelValue`，保证清空后的业务状态同步。
- 插槽与样式分层：结构扩展优先使用 `prefix/suffix` 插槽，样式细化通过 `ui.root/ui.input/ui.clearable` 覆盖，避免直接改内部结构。
- 主题定制优先 `theme`：尺寸、皮肤与全局风格差异优先走 `theme`，局部临时差异再走 `ui`，便于后续主题升级。
