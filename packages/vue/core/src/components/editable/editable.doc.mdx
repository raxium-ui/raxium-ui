# Editable

`Editable` 用于将静态文本切换为就地编辑输入，适合昵称、标题、备注等轻量修改场景。组件基于 Ark UI Editable 构建，支持受控/非受控值、编辑态切换、提交与回退事件，并在 Raxium UI 中扩展了 `theme`、局部 `ui` 和组合插槽能力。

## ARK UI Link

`Editable` 组件族与 Ark UI Editable 一一对应，但属于“部分透传 + Rark 扩展”：

- `EditableProps` 通过 `extends EditableRootBaseProps` 继承 Ark Root 能力，并扩展了 Raxium UI 的 `theme`、`ui`、`class`（以及 `clearable` 字段）。
- `EditableInputProps` / `EditablePreviewProps` 分别 `extends EditableInputBaseProps` / `EditablePreviewBaseProps`，在 Ark 原语基础上封装默认样式；`EditableInput` 额外提供 `clearable` 快捷清空交互。
- `EditableControl`、`EditableEditTrigger`、`EditableSubmitTrigger`、`EditableCancelTrigger` 为 Ark UI 对应子组件导出，可直接组合使用。

Ark UI 文档： [Ark UI Editable (Vue)](https://ark-ui.com/vue/docs/components/editable)

## 最佳实践

- 优先使用 `Editable + EditableInput + EditablePreview` 的固定结构，避免缺失任一子组件导致交互不完整。
- 简单场景使用非受控（`defaultValue`）；需要与外部状态同步或联动校验时再切到受控（`v-model`、`v-model:edit`）。
- 需要明确提交/取消行为时，将 `activation-mode="none"` 与 `EditableControl` + `Submit/CancelTrigger` 组合使用。
- 若启用 `clearable`，建议在 `EditableInput` 上配置，并在 `clearable` 插槽里替换为业务图标以保持视觉一致。
- 对高频编辑输入建议监听 `valueCommit` 做持久化、监听 `valueRevert` 做埋点或回滚提示，而不是仅依赖 `valueChange`。
- 样式定制遵循“先 `theme` 后 `ui/class`”：全局风格走 `theme`，局部布局和微调走 `ui.root/ui.area` 与子组件 `class`。
