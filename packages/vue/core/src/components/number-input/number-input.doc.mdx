# NumberInput

`NumberInput` 用于处理数值输入场景，支持键盘输入、步进调整、范围限制与格式化展示。组件基于 Ark UI Number Input 构建，在根能力透传的基础上扩展了 `showTrigger`、`theme` 与 `ui` 配置，便于统一输入框风格与交互细节。

## ARK UI Link

`NumberInput` 基于 Ark UI Number Input 根能力封装：数值状态、范围约束、步进逻辑、格式化参数等核心能力由 Ark UI 提供；Raxium UI 额外补充了触发器显示开关与主题样式入口。

- **相关文档**: [Ark UI Number Input (Vue)](https://ark-ui.com/vue/docs/components/number-input)
- **透传边界**: 常用 Ark UI Root 参数（如 `modelValue/defaultValue`、`min/max/step`、`locale`、`formatOptions` 等）可直接使用；`showTrigger`、`theme`、`ui.*` 为 Raxium UI 扩展项。

## 最佳实践

- 范围明确的业务（如数量、库存）建议始终设置 `min/max/step`，减少非法值进入后续流程。
- 需要格式化显示时优先使用 `locale + formatOptions`，避免手写字符串格式化导致输入态与展示态不一致。
- 是否自动纠正越界值应根据业务选择：金融/精确录入可将 `clampValueOnBlur` 设为 `false`，由表单校验统一处理。
- 仅在需要鼠标快速调整时开启 `showTrigger`；移动端或紧凑布局中可保持隐藏以降低视觉负担。
- 外部状态同步使用 `v-model` + `update:modelValue`，精细交互追踪再补充监听 `valueChange/valueInvalid/focusChange`。
- 需要外部控制焦点和步进时，可通过组件实例的 `$api` 调用 `focus/increment/decrement/setToMin/setToMax/clearValue`。
